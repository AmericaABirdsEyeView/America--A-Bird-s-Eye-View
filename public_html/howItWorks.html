How it works

Here is a brief overview of how this application works. If you're more technical and want to see the code, you can check out our GitHub account here.


Data Collection

In order to collect the data we need, we employ the Twitter streaming API. We wrote a Python script that initiates a connection to Twitter, and begins downloading tweets that match our requirements. For key terms, we use “obama, barack obama, president barack obama, barack, president, president Obama” and Twitter gives us the tweets that match any of those terms, as they are tweeted. We collect tweets in chunks of 50,000, and collect 5 times a day. This allows us to evenly spread our target of collecting 250,000 tweets per day in such a manner that no one time zone is unfairly represented.


Sentiment Analysis

First, the algorithm goes through every tweet, and separates them by geographic location. This is done by searching for recognizable locations within a tweet's “location” field. The algorithm then builds a training set from the data by gathering all tweets that contain a positive or negative emoticon, and classifying these tweets as positive or negative. From this training set, the algorithm builds a Naïve Bayes Classifier by analyzing the term frequency-inverse document frequency of each term in the tweet vocabulary. The sentiment of each tweet is then evaluated by state and is classified as either “positive” or “negative”. The results of the analysis are then written to a text file and put on our server.


Hubs and Authorities

In order to determine which tweets, hashtags, and URLs are the most popular or are the top “authorities”, the algorithm goes through each tweet and extracts the “retweeted_status” field. From this field the algorithm extracts the IDs of tweets that are being retweeted, and any hashtags or URLs that these tweets contain. Next, the algorithm analyzes all of the extracted retweets, hashtags, and URLs to determine the top 10 most “authoritative” tweets, hashtags, and urls.


User Interface

Our website uses a combination of Twitter Bootstrap, jQuery, and jQuery UI to gain it's aesthetic appeal, but it does so much more than look pretty. It allows users to pick a date that they want to see data for, using a jQuery UI Datepicker. Once the user picks a date, using JavaScript and AJAX we grab the data files corresponding to that date. First, we parse the approval ratings data. We iterate through each state, coloring them corresponding to their approval rating (which is calculated by positive tweets / total number of tweets), setting the state's HTML title property to the approval, and calculating the national approval rating as we go along. Next, we parse the top tweets. We go through the tweet's JSON representation and grab the profile_image_url_https, text, name, and screen_name elements. We use the profile_image_url_https to get the profile picture, the screen name to generate a link to the Twitter profile page, and the name and the text to display. However, before we display the text, we parse it using regular expressions to create links out of the URLs, hashtags, and replies in the text. Once that is done, we parse our top URLs, for these we use the title and URL provided by the Hubs and Authorities algorithm to display the title and a link, and we use Web Backlinks to generate our image. Last, we parse the top hashtags, for these we simply use the hashtag as a title, generate a link to the Twitter search page for that hashtag, and use the same Web Backlinks service to generate the image, because we couldn't think of anything better to display as an image.
